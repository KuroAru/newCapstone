# 체셔의 문제 출제 로직

체셔는 미리 정해진 문제 은행에서 순서대로 퀴즈를 출제합니다.
플레이어가 맞히면 다음 문제로, 틀리면 힌트를 주고 다시 도전하게 합니다.
5문제를 모두 맞히면 퀴즈가 끝납니다.

---

## 문제 출제 전체 흐름

```mermaid
flowchart TD
    subgraph START["① 퀴즈 시작"]
        A["🖱️ 플레이어가 체셔 창문 클릭"] --> B["🦜 앵무새 클릭"]
        B --> C["📋 문제 은행에서\n현재 문제 번호 확인\n정답 수 = 현재 인덱스"]
        C --> D["💬 체셔가 퀴즈 질문을\n한 줄로 출제\n인사·설명 없이 바로 질문"]
    end

    subgraph ANSWER["② 플레이어 답변 입력"]
        D --> E["✏️ 플레이어가\n답을 입력하고 제출"]
        E --> F{입력값이\n있는가?}
        F -- 비어있음 --> G["⚠️ 빈 입력 무시\n다시 대기"]
        G --> E
        F -- 입력 있음 --> H{퀴즈가\n아직 진행 중인가?}
        H -- 이미 완료 --> I["🔒 입력 잠금\n제출 차단"]
        H -- 진행 중 --> J["📤 채점 서버로 전송\n문제 ID + 플레이어 답변"]
    end

    subgraph GRADE["③ 채점 서버 판정"]
        J --> K["⏳ 서버가 답을 확인하는 중\n최대 30초 대기"]
        K --> L{서버 응답}
        L -- 통신 실패 --> M["❌ 오류 안내\n다시 입력 대기로 복귀"]
        M --> E
        L -- 응답 성공 --> N{정답인가?}
    end

    subgraph CORRECT["④-A 정답 처리"]
        N -- 정답 ✅ --> O["🎉 정답 수 +1\n현재까지 n/5 문제 완료"]
        O --> P{5문제\n모두 완료?}
        P -- 아직 --> Q["💬 체셔가 짧게 격려\n다음 문제를 출제"]
        Q --> D
        P -- 완료! --> R["🏁 퀴즈 미션 완료\n체셔가 마무리 인사"]
        R --> S["🔒 입력창 잠금\n퀴즈 UI 종료"]
    end

    subgraph WRONG["④-B 오답 처리"]
        N -- 오답 ❌ --> T{서버가\n힌트를 줬는가?}
        T -- 힌트 있음 --> U["💡 체셔가 힌트를 말함\n서버에서 받은 참고 내용 포함"]
        T -- 힌트 없음 --> V["💬 체셔가 기본 오답 반응\n다시 생각해 봐!"]
        U & V --> W["✏️ 다시 입력 대기\n같은 문제 재도전"]
        W --> E
    end

    subgraph SKIP["⑤ 문제 건너뛰기 선택지"]
        X["➡️ 플레이어가\n다음 문제 버튼 클릭"] --> Y{오답 후\n건너뛰기 대기 중?}
        Y -- 아니요 --> Z["무시"]
        Y -- 예 --> AA["문제 순서 한 칸 밀기\n정답 수는 그대로 유지"]
        AA --> Q
    end

    style START fill:#1e1e2e,stroke:#cba6f7,color:#cdd6f4
    style ANSWER fill:#1e1e2e,stroke:#89b4fa,color:#cdd6f4
    style GRADE  fill:#1e1e2e,stroke:#fab387,color:#cdd6f4
    style CORRECT fill:#1e1e2e,stroke:#a6e3a1,color:#cdd6f4
    style WRONG  fill:#1e1e2e,stroke:#f38ba8,color:#cdd6f4
    style SKIP   fill:#1e1e2e,stroke:#f9e2af,color:#cdd6f4

    style O fill:#0d3320,stroke:#a6e3a1,color:#fff
    style R fill:#0d3320,stroke:#a6e3a1,color:#fff
    style S fill:#3a0000,stroke:#f38ba8,color:#fff
    style U fill:#3a2a00,stroke:#f9e2af,color:#fff
```

## 문제 은행 구조

```mermaid
flowchart LR
    A["📄 TutorQuestionOrder.txt\n문제 출제 순서 목록"] --> B["문제 1번 ID"]
    A --> C["문제 2번 ID"]
    A --> D["문제 3번 ID"]
    A --> E["문제 4번 ID"]
    A --> F["문제 5번 ID"]

    B & C & D & E & F --> G["📚 quiz_bank\n실제 문제·정답 데이터\n서버에서 관리"]

    G --> H["🔍 채점 서버\n플레이어 답변과 정답 비교"]
```

> 문제 순서는 `TutorQuestionOrder.txt` 파일에서 위에서부터 순서대로 읽습니다.
> 정답을 맞힐 때마다 다음 줄로 넘어가고, 건너뛰면 한 칸 밀립니다.

---

## 퀴즈 진행 상태 요약

| 상태 | 의미 | 다음 행동 |
|------|------|----------|
| **답변 대기 중** | 체셔가 질문을 했고 플레이어 입력을 기다림 | 플레이어가 답 입력 |
| **채점 중** | 서버에 답을 보내고 결과를 기다리는 중 | 자동 진행 |
| **정답 후 다음 대기** | 정답 처리 완료, 다음 문제로 넘길 준비 | 체셔 대사 후 자동 출제 |
| **오답 후 재도전 대기** | 틀렸고 힌트를 받은 상태 | 플레이어가 다시 입력하거나 건너뜀 |
| **완료** | 5문제 모두 정답 | 퀴즈 종료, 입력 잠금 |

---

## 건너뛰기 규칙

- 건너뛰기는 **오답을 받은 직후에만** 가능합니다.
- 건너뛰어도 **정답 수는 올라가지 않습니다.**
- 건너뛴 문제 이후로 순서가 한 칸 밀려 다음 문제가 출제됩니다.
- 5문제를 모두 맞혀야 퀴즈가 끝나므로, 건너뛰기를 반복하면 마지막 문제가 계속 반복될 수 있습니다.

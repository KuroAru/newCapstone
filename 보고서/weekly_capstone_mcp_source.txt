3. 주요 수행 내용 (상세)
────────────────────────
[저장소·CI]
• GitHub Actions: C# 구문 검사를 CSharpSyntaxChecker 기반으로 교체하고, Unity가 무시하던 csproj를 .gitignore에서 추적 가능하도록 수정함.
• Python 백엔드 테스트(backend_ai) 린트 오류를 수정하여 CI 통과를 맞춤.

[게임 시스템·씬]
• 아이템: ID 기반 습득 비트마스크 및 습득 추적(ItemAcquisitionTracker, ItemPickup 등) 정비. 체셔 AI(GlobalChatbot)에 인벤토리·진행 상황이 들어가도록 연동.
• 점프스케어(JumpscareManager): 버그 수정, 빌드에서 테스트 씬 제외, 메인 메뉴에서 플레이 가능한 로드 흐름 연결(MainMenu, MainMenuScene).
• 1층 메인 홀(Hall_playerble): 특정 구간에서 적이 항상 나오던 문제 수정. 적 스프라이트가 UI 뒤로 가는 문제를 위해 레이어·정렬(EnemyRenderSorting, EnemySortingUtility, TagManager) 적용.
• Fungus: DialogInput에서 스페이스바로 대사를 넘길 수 있게 함.
• 지도/UI: HideIfFungusBoolTrue 등으로 지도 표시 이슈 조정.

[AI·프롬프트]
• 주방 챗봇(KitchenChatbot): 프롬프트가 연결되지 않던 문제 수정, SonRoomPrompt.txt 추가 및 클릭·버튼 흐름(ClickedBubble, WhenClikcedButton)과 맞춤.

[문서·자산]
• 루트 README.md 작성.

4. 문제점 및 해결 방안
────────────────────────
• 문제: 점프스케어·홀 씬에서 적 출현·렌더 순서·대사 입력 등 플레이 체감 버그가 다수 보고됨.
  해결: JumpscareManager·프리팹·씬을 수정하고, Fungus 입력과 UI/적 레이어를 분리해 재현 환경에서 동작 확인 후 커밋.

• 문제: 주방 챗봇에 신규 프롬프트가 연결되지 않음.
  해결: 리소스(SonRoomPrompt) 추가 및 스크립트에서 로드·전달 경로를 수정.

• 문제: CI에서 C# 프로젝트 무시·린트 실패.
  해결: 워크플로와 .gitignore, 테스트 코드를 정리해 자동 검사가 안정적으로 돌아가도록 함.
